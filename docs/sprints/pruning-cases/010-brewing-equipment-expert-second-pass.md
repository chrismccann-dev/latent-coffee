All headers, the archive, and MCP descriptions written to the measured numbers. (2) **The #521 home-variant row was cut, not relocated** - every clause (location-independent dial logic, home filter compatibility, re-confirm-flow-timing-at-home, paper-matters-less) already exists in sworks.md's intro paragraphs + § SWORKS filter-pairing row, and home water lives in § Location Constraints; nothing unique remained. (3) **T-83 "highest tannin risk - use sparingly" was the one filter-table clause with no home in filters.md** - moved into the DC4 entry's use-case line rather than kept in the op-ref pointer block (single canonical home). (4) **Historical ship records left un-rewritten:** sub-skills-status.md L132 + the Wave-4 kickoff + CR-7 scoping docs still name the old section roster as ship-time facts - archive-layer, not live dispatch surfaces; the four live surfaces (start-brew.md, operational-guide.md ×2, docs.ts) were rewired.
- **Heuristic learned:** **A migration-era "operational reference" that predates its cluster's per-item canonical docs decays into stale label-based duplication - consolidate it to pointers as soon as the per-item docs carry measured data.** The op-ref filter table still said CAFEC T-92 "Slow" / T-83 "Very Slow" while filters.md had measured 80s / 68s; the duplication wasn't just weight, it was *contradiction*. Corollary: when executing a ratified prep report, re-measure its counts against the validation mirror before writing them into headers - prep-report numbers age like doc numbers.
