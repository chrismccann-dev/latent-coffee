To actually shrink a body you **delete** (provenance home = git history + [shipped.md](docs/sprints/shipped.md), per the policy below) or push to an external file (progressive disclosure). | The bootstrap *application* - line counts went UP (brew 210->212) when v0's IR2 said comment-to-save-load; falsified on execution |

A citation that is a rule's evidence/warrant, or a calibration example that IS the lesson (Session 03's docs.ts-vs-brew-import contrast), is load-bearing reference - keep it. | [04](docs/audits/skills/04-architecture-review.md) - the R-rules' inline session-citations are the warrant, not decoration |
| **IR11** | **Cross-skill extraction is `shared-mass x N`, not N alone.** A 2-line blockquote x3 doesn't earn a shared file (run 03 left it inline); a 15-20 line spine x3 does (the review-family). Weigh both dials. | [06-review-family-cross-skill](docs/audits/skills/06-review-family-cross-skill.md) - the review-skill spine across arch/design/improve-skill |

## Decisions

- **IR6 resolution (2026-06-30): no shared-snippet doc.** For ~2-line operator-direct blockquotes, a shared doc behind a `read_doc` pointer adds a hot-path fetch + indirection that costs more than the drift risk on short, stable text. The COLLAPSE cuts shrank the triad's bulk directly (Out-of-scope and MCP-only dropped to one-liners / deletions); the Operator-direct blockquote stays inline and identically worded across the three so drift is visible. The N=3 "extract on the third copy" dial is overridden here by the indirection-cost judgment (the architecture-review considered-and-kept discipline).
- **Provenance policy (2026-06-30): delete, keep skills lean.** Pure historical provenance (PR#, dates, incident narratives) is deleted from skill bodies, not inline-commented. git history + [shipped.md](docs/sprints/shipped.md) are its home. Chris's call; diverges from Latent's cite-inline doc culture deliberately, because a skill body is loaded into a working session and every token competes with the task. (Bounded by IR10: a citation that *is* a rule's evidence/lesson stays.)
- **Review-skill invocation (2026-06-30): user-invoked.** `architecture-review` + `design-review` set `disable-model-invocation: true` and strip their model-facing descriptions. Chris runs them on a routine + calls them deliberately, so natural-language triggering wasn't earning the always-loaded description. `improve-skill` was already user-invoked - the three review skills are now a consistent user-invoked family.
- **Review-skill spine (2026-06-30): extracted.** The ~15-20 line shared doctrine (IR11) moved to [.claude/skills/review-skill-spine.md](.claude/skills/review-skill-spine.md); arch-review / design-review / improve-skill each point at it and keep only their domain rules (R / D / IR). Chris's call. The per-skill "two guardrails" preview blocks (which duplicated both the spine and each skill's own later steps) were removed; the domain rules stay intact, each skill still reads standalone.
