# 0003 — Anchor profile / anchor confidence vocabulary canonicalization

The upstream input to V1 roast design is named **anchor profile** (formal noun) or **anchor** (conversational shorthand), with a six-level **anchor confidence** ladder (High / Medium-High / Medium / Medium-Low / Low / Very Low) and a distinct **Deferred** field state for lots without an archive anchor yet. Alternatives considered: keep the polysemy (working prose uses "anchor profile" / "anchor roast" / "reference profile" interchangeably), promote "reference profile" (parallels the brewing-side reference brew), or canonicalize "anchor roast" (matches the inventory document's column header). Canonicalizing **anchor profile** wins because "reference profile" collapses with the lot-close **reference roast** designation (different lifecycle scope, different referent), "anchor roast" is schema-column-influenced not prose-natural, and the bare conversational **anchor** is already the operationally dominant form. The decision pairs the upstream **anchor profile** (parametric reference, source artifact) explicitly with the downstream **carry-forward learnings** (propositional knowledge, distilled prose) — design opens with the anchor, interpretation closes with the carry-forward; one-to-many relationship. The 6-level confidence ladder is locked because it appears verbatim in the inventory document already; the distinct **Deferred** field state (vs Very Low) is locked because the two operational responses differ (Deferred → halt V1; Very Low → proceed with wide exploratory spread). Reversal cost is moderate (vocabulary changes propagate to inventory document field semantics + prompt language); the round-2 cross-party audit surfacing of "the glossary crystallized the downstream concept and left the upstream one undefined" is the load-bearing rationale.
