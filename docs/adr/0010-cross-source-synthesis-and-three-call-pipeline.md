# 0010 — Cross-source synthesis + three-call pipeline + mobile short-form

The synthesis pipeline now (a) reads both `brews` AND `roast_learnings` rows on the terroir, cultivar, and roaster adapters — joined through `green_beans.{terroir_id, cultivar_id}` (Latent path on the roaster adapter pulls all roast_learnings unconditionally) — and integrates both vocabularies into ONE unified narrative voice per a top-of-rules SHARED_RULES directive (anti-pattern flagged: appended brewing-side / roasting-side sections); (b) extends the ADR-0002 two-call shape to **three sequential Sonnet 4.6 calls** per regeneration (raw → humanizer → short-form digest), accepting ~3x latency + credit per regeneration on the same lazy-trigger bounding (page-visit only, on `brewCount` or `synthesis_input_max_updated_at` delta); (c) persists a `short_form_capsule text` column on each of the 4 cache surfaces and renders it below the `md:` breakpoint via SynthesisCard, with long-form fallback when the short-form is NULL. Two deliberate scoping decisions live alongside: the **process adapter is deferred from SYN-6** (signature-method join surfaces zero rows today — `green_beans` has no `signature_method` column, and the 2 brews with `brews.signature_method` populated both have `green_bean_id=NULL`; modifier-level cross-source through reference-roast brews is structurally heavier and not warranted at current corpus size), and **`is_process_dominant` is wired as a prompt-level flag** through `formatLearningRow` rather than an adapter-level weight tier — the SHARED_RULES directive instructs the model to attribute observations on heavy-process brews to the process layer rather than letting them generalize to the cultivar / terroir / roaster anchor. Migration 065 adds `short_form_capsule` + `synthesis_input_max_updated_at` to terroirs / cultivars / roaster_syntheses / process_aggregation_syntheses (8 column adds, idempotent, no backfill — NULL means "stale, regenerate on next page visit"). Scenario 1 (anchor FK reassigned post-arbiter) and scenario 4 (registry rich-anchor code edits) remain manual-Regenerate-button territory; neither is frequent enough today to warrant `synthesis_input_max_updated_at` tracking. Re-test the three-call cost shape if Chris's monthly Anthropic bill changes materially or if the short-form digest call's failure rate exceeds 5%.
