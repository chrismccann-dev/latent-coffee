# Two-axis brewing framework as a deliberate compression of WBC's five-axis taxonomy

Brewing research at Latent is grounded in the WBC competition corpus (102 recipes, 2022-2025), which describes brews along five foundational control axes (Extraction Intensity / Time Distribution / Physical System / External Control / Output Selection) and roughly eight named strategy families. Mirroring that shape in the app would require a multi-axis brew row, a much larger canonical taxonomy per axis, and per-axis pickers in every author surface. Instead we compress to two axes: one canonical **Extraction Strategy** (one of 6 values: Suppression / Clarity-First / Balanced Intensity / Full Expression / Extraction Push / Hybrid) plus zero-or-more layered **Modifiers** (4 canonical types post-v8.5: output_selection / inverted_temperature_staging / aroma_capture / role_based_pulse).

The compression is a **learning-stage** choice, not a permanent one. Three things make it the right call now: (1) many WBC subtypes — thermal mixing, tool-assisted moves, intentional roast-blending — are not in Chris's active practice yet, so taxonomic precision against them buys nothing; (2) WBC's Physical System axis (Structural Systems family) is explicitly out of scope for single-origin home brewing; (3) the modifier system gives **extensibility** — Chris can represent a WBC move he isn't actively pursuing without committing it to first-class status. Promotion path is explicit: a modifier becomes a strategy when sustained usage + cross-coffee pattern emerge (v8.4 promoted the Immersion modifier → Hybrid strategy when SWORKS Bottomless Dripper office work and selective Hario Switch home work crossed the threshold). The 2-axis shape itself may expand to 3-5 axes in the future if Chris's brewing scope broadens.

Trade-off accepted: less taxonomic precision against WBC source material vs simpler app representation, a more tractable picker surface, and clearer single-strategy intent per brew. Cross-cutting WBC moves that don't promote to canonical modifiers (water strength / agitation taper / filter behavior / pre-brew conditioning / time-distribution templates beyond Role-Based Pulse) persist as doc-layer reference patterns in [docs/skills/wbc-brewing-archivist/cluster/wbc-reference.md](../skills/wbc-brewing-archivist/cluster/wbc-reference.md) (migrated from `docs/brewing/wbc-reference.md` in Wave 2 PR 1, 2026-05-26) and via free-text `strategy_notes` on the brew row when they're doing strategic work on a given coffee.
