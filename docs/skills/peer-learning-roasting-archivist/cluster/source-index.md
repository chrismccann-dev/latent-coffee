## Source-type tier

Peer-roaster content is weighted differently by source type. Roughly:

- **Tier A: live-roasting livestream / video** — highest weight, lets Chris see decision moments + numbers + commentary together. Dongzhe Yunnan-Hatchi livestream is the canonical example.
- **Tier B: long-form claude.ai discussion (chat history with Chris)** — high weight; same person, less structured. Source for Dongzhe's pre-livestream Reference Roast Target + Core Principles.
- **Tier C: written blog post / static doc** — medium weight; framing is theirs, but no real-time decision context.
- **Tier D: short-form Instagram / Twitter / forum post** — low weight; usually loses too much context for actionable integration.

Source tier doesn't gate integration — Chris decides what's worth pulling. The tier is a hint about confidence calibration when integrating.
